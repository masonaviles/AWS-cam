### Setting Up the DynamoDB Table

- **Purpose**: Store the website view count.
- **Steps**:
	- Access the DynamoDB service in the AWS console.
	- Click "Create table."
	- Name the table (e.g., "cloud resume-test").
	- Define the partition key as "id" (lowercase).
	- Optionally, add a tag for project organization (e.g., key: "project", value: "Cloud Resume Challenge").
	- Click "Create table."
	- ![](attachments/Pasted%20image%2020250109151907.png)
- **Adding an Item**:
	- Once the table is created, click "Actions" and select "Explore items."
	- Click "Actions" again and choose "Create item."
	- Set the partition key "id" to 1.
- Add a new attribute:
	- Name: "views" (lowercase)
	- Type: Number
	- Value: 1
	- Click "Create item."
	- ![](attachments/Pasted%20image%2020250109152117.png)

### Building the Lambda Function

- **Purpose**: Act as an API to fetch view count data from DynamoDB, avoiding direct calls from the website.
- **Steps**:
	- Open the AWS Lambda console.
	- Click "Create function."
	- Name the function (e.g., "cloud resume-API").
	- Choose your preferred runtime (e.g., Python 3.12).
	- For execution role, create a new role or select an existing one.
- Under "Advanced settings":
	- Enable "Function URLs" for public URL access to the function.
	- Keep "Auth type" as "None," meaning anyone can call the API (for now).
	- **Important**: Cross-Origin Resource Sharing (CORS) will be configured later to whitelist only the specific domain of your website for security.
	- Click "Create function."
- **Initial Behavior**: The created function will have default code that returns "hello from Lambda" when invoked.
- **Next Step**: Modify the Lambda function's code to interact with the DynamoDB table and retrieve the view count.
