
# AWS Cloud Resume Challenge

## Key Steps and Ideas:

1. **Preparing the Website:**

	- The video assumes you already have a static website built with HTML, CSS, and potentially JavaScript files.
	- Ensure your website is functioning correctly locally before proceeding.

2. **Creating an S3 Bucket:**

	- In the AWS console, create a new S3 bucket with a globally unique name.
	- **Important:** Keep the bucket private by blocking all public access.
	- ![](attachments/Pasted%20image%2020250109145201.png)

 3. **Uploading Website Files:**

	- Upload all your website files (HTML, CSS, images, etc.) to the newly created S3 bucket.
	- ![](attachments/Pasted%20image%2020250109145218.png)
	- Enabled static website hosting
	- Defined out index.html
	- ![](attachments/Pasted%20image%2020250109145556.png)

4. **Setting up CloudFront:**

	- Create a new CloudFront distribution.
	- ![](attachments/Pasted%20image%2020250109145715.png)
	- Configure the distribution to use your S3 bucket as the origin.
	- Select the correct S3 bucket from the list.
	- **Origin Access Control:** Since the S3 bucket is private, configure an Origin Access Control (OAC) to grant CloudFront permission to access the bucket's content.
	- Copy the provided policy and paste it into the bucket policy within the S3 bucket's permissions settings.
	- ![](attachments/Pasted%20image%2020250109145858.png)
	- Enforce HTTPS Only: Ensure your website is delivered securely by forcing HTTPS connections.

5 . **Configuring Default Root Object:**
	- Within the CloudFront distribution settings, define the "default root object" as "index.html" to ensure the correct file loads when the website is accessed.

6 . **Custom Domain and SSL Certificate:**
	- **Route 53:** Acquire a custom domain (e.g., resume.cameronxdx.cloud) using Route 53 or other domain registrars.
	- ![](attachments/Pasted%20image%2020250109150140.png)
	- **CloudFront Alternate Domain Name:** Configure CloudFront to use your custom domain as an alternate domain name (CNAME).
	- ![](attachments/Pasted%20image%2020250109150245.png)
	- **AWS Certificate Manager (ACM):** Request a public SSL certificate through ACM for your custom domain. Ensure it covers the desired subdomains using a wildcard certificate (e.g., *.cameronxdx.cloud).
	- **Link SSL Certificate to CloudFront:** Select the issued SSL certificate from ACM within your CloudFront distribution settings.
	- ![](attachments/Pasted%20image%2020250109150319.png)

7 . **Route 53 Record Configuration:**
	- Create a new 'A' record within Route 53 for your subdomain (e.g., resume.cameronxdx.cloud) pointing to the CloudFront distribution.
	- ![](attachments/Pasted%20image%2020250109150535.png)
	- Select the "Alias" option to route traffic directly to the CloudFront distribution.

 8.  **DNS Propagation:**
	- Allow some time for DNS changes to propagate across the internet before your website becomes accessible via the custom domain.

8. **Verification:**

	- Test the website by accessing it using the custom domain (e.g., https://resume.cameronxdx.cloud).
	- Verify that the website loads correctly and that the SSL certificate is working by checking the lock icon in the browser's address bar.

**Important Concepts:**

- **S3 (Simple Storage Service):** An object storage service used to store website files.
- **CloudFront (Content Delivery Network):** A service that speeds up content delivery by caching it at edge locations globally.
- **Origin Access Control:** A security mechanism that allows CloudFront to access private S3 buckets.
- **SSL Certificate:** Encrypts communication between the website and visitors, ensuring data security.
- **Route 53:** A DNS service used to manage domain names and route traffic.
- **DNS Propagation:** The time it takes for DNS changes to be reflected across the internet.
